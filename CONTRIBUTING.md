\# Contributing to Software Version Reporter Add-on



Thank you for your interest in contributing! Your help improves the add-on for everyone.



\## How to Contribute



\### Reporting Issues



\- Please open issues on the GitHub repository with a clear description of the problem or feature request.

\- Provide steps to reproduce bugs whenever possible.



\### Submitting Pull Requests



1\. Fork the repository.

2\. Create a topic branch for your feature or bugfix.

3\. Write clean, well-documented code following ZAP and Java conventions.

4\. Test your changes thoroughly.

5\. Submit a pull request explaining what you have implemented or fixed.



\### Coding Style



\- Follow the OWASP ZAP coding style (similar to standard Java conventions).

\- Write modular, maintainable and well-tested code.

\- Ensure all new features and fixes are backward compatible.



\### Testing



\- Run the Gradle build tasks to verify your changes:

./gradlew clean build



text

\- Verify the add-on loads correctly in ZAP and performs as expected.



\### Logging



\- Use existing logging facilities (`org.apache.logging.log4j.Logger`) to log errors and debug information.

\- Do not expose sensitive information in logs.
